\section{Complete Experiment Artifacts}
\label{app:experiments}

This appendix gives the full experiment trail behind the claim-specific
evidence structure in the main text. The purpose is not to add a second,
independent argument after the paper's conclusion has already been stated.
Rather, the appendix makes the operational details visible: which datasets were
used, which baselines define the relevant failure modes, how the primary
measures were computed, and where the diagnostic evidence should be read as
supporting or boundary evidence rather than as proof.

The organisation follows the same logic as the main paper. We first specify the
datasets and baseline methods, because the scope of a practical-use claim is
determined by the regimes in which it is tested. We then report the four primary
measures for public and controlled datasets, because a single score would hide
the distinction between marginal fidelity, row reuse, dependence preservation,
and downstream utility. Finally, we include the mechanism and boundary
diagnostics for NCA blocks, decoder probabilities, and frozen-Isomap insertion
stress. In other words, the appendix is meant to be read as an audit record for
the allowable conclusion, not as an attempt to convert a bounded empirical
claim into a universal benchmark claim.

\subsection{Datasets And Methods}

Table~\ref{tab:datasets} defines the empirical scope of the paper. The public
datasets are intentionally heterogeneous in size, domain, categorical
composition, and missingness, while the controlled datasets isolate specific
failure modes that are hard to attribute cleanly in public benchmarks. This
choice creates a mixed evidence base: public datasets support practical
workflow relevance, whereas controlled datasets make particular boundary
conditions observable under known structure.

The dataset selection is therefore structured to cover distinct failure modes
rather than to maximise the number of benchmarks. Adult acts as the primary
running example because it is a familiar, moderately sized, mixed-type dataset
with interpretable categorical and numerical dependencies. Titanic is used for
small illustrative figures, while the medical, business, and controlled
datasets test whether the same diagnostics remain legible across different
regimes. Covertype is included to cover a larger public source table without
changing the paper into a scale study: the full prepared table is retained, but
the notebook workflow uses a deterministic cap when running repeated
diagnostics. This design keeps the paper method-focused rather than turning it
into a domain-specific benchmark study.

Table~\ref{tab:methods} records the comparison methods used throughout the
experiments. The baselines are deliberately simple because the central claim is
not that DataFrameSampler dominates specialised synthetic-data systems; it is
that an inspectable low-setup generator can be useful when ordinary example
data are needed. Row bootstrap is included even though it is not a valid
privacy-preserving generator, because it exposes the misleading behaviour of
methods that obtain strong distributional or utility scores by direct row
reuse. Independent-column and stratified empirical samplers define the opposite
failure mode, in which marginal distributions can look reasonable while
cross-column structure is weakened.

\subsection{Formal Definitions of Performance Measures}
\label{app:measure-definitions}

This section defines the reported performance and diagnostic measures. Let
$X=\{x_i\}_{i=1}^n$ denote the real dataframe rows after the metric-specific
preprocessing step, and let $\widetilde{X}=\{\widetilde{x}_j\}_{j=1}^m$ denote
the generated rows. When a target column is present, nearest-neighbour,
discrimination, and utility tests remove that target from the feature matrix
before fitting the corresponding evaluator.

\paragraph{Nearest-neighbour distance ratio.}
Let $d(\cdot,\cdot)$ be Euclidean distance after the shared preprocessing used
by the evaluator. For each real row, define its leave-one-out nearest-neighbour
distance
\[
  r_i = \min_{\ell\ne i} d(x_i,x_\ell),
\]
and for each generated row define its nearest real-row distance
\[
  \widetilde{r}_j = \min_i d(\widetilde{x}_j,x_i).
\]
The reported ratio is
\[
  \mathrm{NNRatio}
  =
  \frac{m^{-1}\sum_{j=1}^m \widetilde{r}_j}
       {n^{-1}\sum_{i=1}^n r_i}.
\]
Values much below one indicate generated rows that are closer to the training
set than real rows are to one another. The suspiciously-close rate is
\[
  m^{-1}\sum_{j=1}^m
  \mathbf{1}\!\left[
    \widetilde{r}_j < Q_{0.05}(\{r_i\}_{i=1}^n)
  \right],
\]
where $Q_{0.05}$ is the real leave-one-out 5th percentile.

\paragraph{Real-versus-synthetic discrimination.}
The discriminator test fits a classifier to distinguish real rows from
generated rows after the same feature preprocessing. Real rows receive label
1 and generated rows label 0. The reported discrimination accuracy is the
held-out accuracy of this classifier. The privacy-style discrimination score is
a bounded descriptive transformation,
\[
  \mathrm{DiscScore}
  =
  \max\{0,\,1-2|\mathrm{Acc}-0.5|\},
\]
so that chance-level accuracy receives score one and perfectly separable data
receive score zero. This is not a formal privacy measure.

\paragraph{Utility lift.}
Let $M$ be the fixed downstream predictive model used by the evaluator and let
$S(\cdot)$ be its held-out score on real test rows. The real-only baseline
trains $M$ on real training rows, giving score $S_{real}$. The augmented model
trains on real training rows plus generated rows aligned to the same schema,
giving score $S_{aug}$. Utility lift is
\[
  \mathrm{UtilityLift}=S_{aug}-S_{real}.
\]
Positive values mean that generated rows improved this particular held-out
predictive task under the fixed evaluator; they do not imply general
downstream usefulness.

\paragraph{Distributional similarity.}
For a numeric column, histogram overlap is computed from normalised histograms
$p_b$ and $\widetilde{p}_b$ over common bins:
\[
  \mathrm{Overlap}=\sum_b \min(p_b,\widetilde{p}_b).
\]
For a categorical column with empirical frequency vectors $p$ and
$\widetilde{p}$ over the union of observed categories, total variation is
\[
  \mathrm{TV}(p,\widetilde{p})
  =
  \frac{1}{2}\sum_c |p_c-\widetilde{p}_c|,
\]
and the Jensen--Shannon divergence is
\[
  \mathrm{JSD}(p,\widetilde{p})
  =
  \frac{1}{2}\mathrm{KL}(p\|a)
  +
  \frac{1}{2}\mathrm{KL}(\widetilde{p}\|a),
  \qquad
  a=\frac{1}{2}(p+\widetilde{p}).
\]
The aggregate distribution similarity score averages the available components
$\mathrm{Overlap}$, $1-\mathrm{JSD}$, and $1-\mathrm{TV}$.

\paragraph{Association difference.}
For mixed-type dependence diagnostics, the evaluator builds an association
matrix $A(X)$ using Pearson correlation for numeric--numeric pairs, Cramer's
$V$ for categorical--categorical pairs, and correlation ratio for mixed
numeric--categorical pairs. The reported mean association difference is the
mean absolute upper-triangular difference:
\[
  \frac{2}{d(d-1)}
  \sum_{1\le a<b\le d}
  |A(X)_{ab}-A(\widetilde{X})_{ab}|.
\]

\paragraph{Decoder calibration.}
For a categorical decoder with class-probability vector $p_i$ and true class
$y_i$, negative log loss is
\[
  -\frac{1}{n}\sum_i \log p_i(y_i),
\]
and the multiclass Brier score is
\[
  \frac{1}{n}\sum_i \sum_c \left(p_i(c)-\mathbf{1}[y_i=c]\right)^2.
\]
Expected calibration error partitions top-class confidence values into fixed
bins $B_b$ and reports
\[
  \mathrm{ECE}
  =
  \sum_b \frac{|B_b|}{n}
  \left|
    \mathrm{acc}(B_b)-\mathrm{conf}(B_b)
  \right|.
\]
These quantities diagnose the probabilistic inverse map; they do not establish
that generated categories are calibrated draws from the true data-generating
distribution.

\paragraph{Frozen-Isomap insertion stress.}
For a generated or held-out point $x$, let $g_i(x)$ be its graph-geodesic
distance to training row $x_i$ after attaching $x$ to the frozen Isomap
neighbourhood graph, and let $e_i(z)$ be its Euclidean distance to the embedded
training point after out-of-sample Isomap transformation. The normalised
pointwise insertion stress is
\[
  s(x)
  =
  \sqrt{
  \frac{\sum_i (g_i(x)-e_i(z))^2}
       {\sum_i e_i(z)^2}
  }.
\]

\paragraph{\textbf{Convex-hull membership.}}
Convex-hull membership is tested in the fitted DataFrameSampler latent space,
not in the raw mixed-type dataframe. Let
$Z=\{z_i\}_{i=1}^n\subset\mathbb{R}^d$ be the training latent rows and let
$z\in\mathbb{R}^d$ be the latent representation of a generated row. The point
$z$ is inside the convex hull of the training set if and only if it can be
written as a convex combination of the training latent rows:
\[
  z = \sum_{i=1}^n \alpha_i z_i,
  \qquad
  \alpha_i\ge 0,
  \qquad
  \sum_{i=1}^n \alpha_i = 1.
\]
Operationally, the experiment asks whether there exists a vector of weights
$\alpha=(\alpha_1,\ldots,\alpha_n)$ satisfying these linear constraints. This
is a feasibility problem rather than an optimisation problem: the actual
objective value is irrelevant, because any feasible set of non-negative weights
summing to one proves that the point can be reconstructed as an average of
observed latent rows. In implementation, the equality constraint
$Z^\top\alpha=z$ is combined with the simplex constraint
$\mathbf{1}^\top\alpha=1$ and non-negativity bounds $\alpha_i\ge0$, and passed
to a linear-programming feasibility solver. If the solver finds feasible
weights, the generated point is counted as in-hull. If no feasible weights are
found, the point is counted as out-of-hull.

Before solving this feasibility problem, the training latent matrix is reduced
to its numerical rank with a deterministic singular-value decomposition. This
rank reduction removes redundant directions caused by one-hot encodings,
constant columns, or other linear dependencies, while preserving the affine
convex-combination question in the span of the training data. The out-of-hull
label is therefore relative to the learned latent representation and the fitted
training rows: it does not mean that the decoded mixed-type row violates a raw
feature range, nor does it certify semantic novelty. It only states that the
latent generated point cannot be expressed as a convex average of the fitted
training latent rows under the numerical feasibility tolerance. The out-of-hull
acceptance rate then counts the fraction of these out-of-hull generated points
whose frozen-Isomap insertion stress is no larger than the held-out real 95th
percentile.

\subsection{Primary Four-Measure Summary}

Table~\ref{tab:main-measures} reports the compact version of the main
evaluation. The four measures have different evidential roles. The
nearest-neighbour distance ratio and suspiciously-close rate are boundary
checks for row reuse or pathological proximity; the discriminator is a
two-sample proxy that asks whether synthetic rows are easily separated from real
rows; utility lift asks whether generated rows improve a downstream held-out
task; and histogram or categorical similarity measures summarise marginal
agreement. These measures are not interchangeable, and therefore disagreement
between them is informative rather than a nuisance.

More precisely, a method can preserve histograms while breaking dependencies,
as independent-column sampling often does, or it can obtain strong utility
numbers while copying rows, as row bootstrap can do. DataFrameSampler is
therefore evaluated as a balanced example generator: it is useful when several
diagnostics are acceptable at the same time, not when one favourable number is
selected in isolation. This is the reason the table includes simple baselines
beside the sampler, even when those baselines are not intended as deployable
synthetic-data methods.

The starter utility-cost frontier in

only produce plausible rows; it must also have a setup and runtime profile that

\subsection{Controlled Synthetic Regimes}

Controlled datasets are included because real benchmarks often make failure
attribution ambiguous. If a method performs poorly on a public medical or
business dataset, it is usually unclear whether the problem is missingness,
cardinality, class imbalance, dependence structure, or an implementation
choice. The controlled regimes therefore introduce known dependency patterns,
rare categories, high-cardinality categorical variables, and a deliberately
sensitive identifier-like column, so that particular weaknesses can be observed
without treating every public-dataset result as a black box.

Table~\ref{tab:synthetic-controlled-datasets} lists these regimes, and
Figure~\ref{fig:appendix-synthetic-controlled-similarity} shows the same
primary measures under controlled conditions. The key interpretation is
comparative. Row bootstrap represents a high-fidelity but unsafe boundary
reference, independent-column sampling represents a marginally plausible but
structurally weak reference, and DataFrameSampler is evaluated by whether it
preserves some useful structure without collapsing into either degenerate
case. Table~\ref{tab:synthetic-controlled-results} then reports focused
quantities such as category coverage, rare-category preservation, association
difference, and exact sensitive-value overlap.

\subsection{Mechanism, Decoder, and Sensitivity Diagnostics}

The mechanism diagnostics address a narrower question than the primary
similarity and utility measures. They do not ask whether the final generated
rows are useful; they ask whether the supervised latent block construction is
doing measurable work. For each categorical column, the validation compares
held-out prediction from the learned NCA block against a majority-class
reference and a same-width PCA projection of the same context. Positive lift
over these references is supporting evidence for the NCA mechanism, whereas
weak or negative lift means the mechanism should be treated as plausible but
not supported for that dataset or column family.

\subsection{Secondary Class-Rebalancing Diagnostic}

The class-rebalancing diagnostic is included because local displacement
generation is adjacent to oversampling methods, especially SMOTE-style
interpolation. We keep the diagnostic deliberately small. It uses only selected
binary target datasets, one fixed train/test protocol, and one fixed rebalance
target: generate enough minority-class rows to bring the real training split to
parity. DataFrameSampler is fit on minority-class feature rows only, so the
generated examples are traceable analogies among observed minority examples.
The comparison to a SMOTE-family reference is reported when the optional
\texttt{imbalanced-learn} dependency is available; it uses SMOTENC for
mixed-type feature tables and ordinary SMOTE for numeric-only feature tables.

\subsection{Frozen-Manifold And Extrapolation Diagnostics}

The manifold validation experiment tests a more specific diagnostic claim:
some generated latent points may lie outside the convex hull of the training
latent matrix while still fitting a frozen Isomap geometry with low insertion
stress. The convex-hull test is carried out in the same DataFrameSampler latent
space used by the generator, after affine-rank reduction, and the Isomap model
is fitted only on training rows. This frozen protocol is important because
refitting the manifold after inserting generated rows would allow the reference
geometry itself to adapt to the samples being evaluated.

The generated points are compared with held-out real points, latent
interpolation, and latent bootstrap. Held-out real points define the stress
baseline, interpolation defines the in-hull or near-in-hull reference, and
bootstrap exposes the effect of reusing latent training rows. The decisive
quantity is not the raw out-of-hull rate alone, because extrapolation without
manifold consistency is not useful. It is the conditional comparison between
out-of-hull generated stress and the held-out real stress distribution.

\subsection{Planned Extensions And Claim Boundaries}

Table~\ref{tab:ablations} summarises which mechanism checks are already
implemented and which remain planned. This table is included in the appendix
because it prevents a common interpretive error: a transparent mechanism is not
the same thing as a validated mechanism. The current evidence can support
practical-use and exploratory mechanism conclusions, but it does not yet
support a broad causal statement that each component is necessary or optimal.

The main unresolved extensions are therefore direct ablations of the generator
and decoder choices under matched capacity and broader regime analysis. In
particular, a stronger future study would vary the NCA iteration count, replace
transport with interpolation under otherwise identical decoding, compare
calibrated and uncalibrated decoders under the same sampling budget, and group
results by dataset descriptors such as cardinality, sparsity, missingness, and
sample size. Until those studies are complete, the appendix keeps the boundary
visible: the method is inspectable and empirically useful in the studied
settings, but its mechanisms remain partially supported rather than fully
established.
