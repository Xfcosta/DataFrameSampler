inspectable generator for mixed-type examples, not when they need an

runtime and memory measurements, and controlled-regime figures are placed in

\subsection{Evaluation Focus}

The evaluation uses Adult Census Income and Titanic as public mixed-type
examples and four deterministic controlled datasets as boundary tests. For each
dataset, we run DataFrameSampler with default, manual, and fixed LLM-style
configurations, plus simple references: row bootstrap, independent columns,
Gaussian-copula empirical sampling, and target-stratified column sampling where
a target exists. The complete tables are deliberately separated from the main
text so that the paper does not become a benchmark catalogue. The main summary
uses four measures: a nearest-neighbour distance test, a discrimination test, a
utility lift test, and a distribution similarity test. These tests deliberately
separate three ideas that are often conflated: rows should not be suspiciously
close to individual source rows, generated rows should not be trivially
distinguishable from real rows, generated rows should improve a held-out
real-data task if they are useful, and feature distributions should remain
similar enough for example-data workflows.

\subsection{Illustrative Results}

The current experiment scaffold includes Adult Census Income and Titanic as
initial mixed-type datasets. For each dataset, we compare simple baselines
against three DataFrameSampler configurations: default settings, a manual
configuration, and a fixed LLM-style configuration recorded in the notebook for
reproducibility. The LLM-style condition should be read as an experiment proxy
for the kind of structured recommendation returned by the LLM interface; the
notebooks do not call an external API during execution.

\subsection{Controlled Synthetic Regimes}

To separate mechanism behaviour from the idiosyncrasies of public datasets, we
also construct four deterministic synthetic tables. These datasets are not used
as evidence that the method handles all real deployments. Their role is to
create known stressors: numeric variables with embedded categorical columns,
high-cardinality categorical values, deliberately rare categories, and a
sensitive identifier column. The controlled identifier dataset is particularly
important for bounding the anonymization claim: without the anonymization layer,
all compared generators can re-emit exact source identifiers because the
identifier is part of the source schema.

\begin{figure*}[t]
\centering
\includegraphics[width=\textwidth]{synthetic_category_stress.pdf}
\caption{Categorical stress tests on the controlled synthetic datasets. The
coverage and rare-preservation metrics make high-cardinality and rare-category
failure modes visible without relying on a single aggregate score.}
\label{fig:synthetic-category-stress}
\end{figure*}

The controlled results show the intended interpretation pattern. Row bootstrap
is a useful reference because it can score well by reusing rows; independent
columns can preserve marginals while breaking dependencies; and the controlled
identifier dataset shows why anonymization must be treated as a separate
pre-generation step rather than inferred from distributional similarity.
