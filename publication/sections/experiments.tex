\section{Experiments}
\label{sec:experiments}

The experiments instantiate the evidence framework in
Section~\ref{sec:evaluation-framework}. The primary comparisons ask whether
generated rows avoid trivial reuse, preserve marginal and dependence structure,
and provide useful augmentation for held-out real tasks. The mechanism
diagnostics ask whether the supervised NCA blocks and decoders behave as the
method interpretation suggests. The boundary diagnostics ask whether local
transport and default parameter choices remain stable enough for the restricted
practical-use claim.

The prepared public dataset suite is designed for coverage rather than
leaderboard breadth. Adult is the running example, Titanic is a small
illustrative case, Bank Marketing provides a mid-sized business benchmark, the
medical datasets exercise small clinical-style regimes, and Covertype adds a
larger public source table with continuous terrain measurements and categorical
terrain context. The repeated diagnostics use deterministic row caps so that
notebook execution remains practical.

Current comparisons include DataFrameSampler plus row bootstrap, independent
columns, Gaussian-copula empirical sampling, and target-stratified column
sampling where a target exists. These baselines define concrete failure modes:
row reuse, broken dependencies, and low-cost marginal sampling. They are not
intended to exhaust the tabular-generation literature.

\subsection{Where the Proposed Method Is Preferred}

The intended preference regime is practical rather than universal. Compared
with row bootstrap, the method is preferred when copying complete records is an
unacceptable default behaviour. Compared with independent column sampling, it
is preferred when cross-column dependence should remain visible in examples.
Compared with heavier tabular generators, it is preferred when a user needs a
small scikit-learn-style object whose fitted latent blocks, neighbour chains,
decoder probabilities, runtime, and failure modes can be inspected directly.
The experiments therefore report where these advantages appear, and they avoid
claiming superiority on datasets or metrics where the evidence does not
support it.

The three named DataFrameSampler setups are shown on one representative
dataset, Adult Census Income. This keeps setup choice separate from the main
cross-dataset evidence: fast, default, and accurate are practical operating
points, not three additional methods to rank across every benchmark.

\subsection{Secondary Diagnostic: Class Rebalancing}

Because the transport operator is adjacent to oversampling-style local
augmentation, we include a limited class-rebalancing diagnostic. This is a
secondary use case, not the paper's primary claim. On selected binary target
datasets, the protocol splits real data into train and held-out test rows,
identifies the minority class in the training split, and adds synthetic
minority rows until the training split reaches parity. DataFrameSampler is fit
only on minority-class feature rows, and generated rows are then labelled as
the minority class. The same held-out real test split is used to compare a
classifier trained on real rows alone with classifiers trained on real plus
DataFrameSampler-balanced rows, real plus a SMOTE-family reference when the
optional dependency is available, and real plus stratified empirical minority
rows. The reference uses SMOTENC when categorical features are present and
ordinary SMOTE for numeric-only feature tables.

The reported metrics are balanced accuracy, macro F1, minority recall, and
binary PR-AUC where applicable. These quantities are minority-sensitive, but
they do not convert the method into a general imbalance-learning optimizer.
They test only whether traceable minority examples can be useful under one
fixed rebalance protocol.

\subsection{Comparison with High-Capacity Generative Models}

We include CTGAN as an optional Adult-only reference model because modern deep
tabular generators are an important design point for reviewers and users.
This comparison is deliberately limited. CTGAN is not treated as a target to
beat, and we do not perform broad hyperparameter optimization. Instead, it is
used to locate DataFrameSampler relative to a model with a global adversarial
training objective, a heavier dependency stack, and lower decomposition into
inspectable components. The same primary metrics are reported for the
reference run:
distributional score, real-versus-synthetic discrimination, utility lift,
runtime, and memory.

Table~\ref{tab:deep-reference-comparison} reports the capped Adult reference
run. CTGAN attains a distributional score close to the default
DataFrameSampler run on the same dataset, but the discriminator separates
CTGAN samples from real rows more easily, and the measured utility lift is
negative in this run. This is not a general CTGAN conclusion: the model was not
tuned, and only one representative dataset is used. The result instead
illustrates the intended comparison role. A high-capacity model can occupy a
nearby fidelity region while offering a different tradeoff: a global
adversarial objective, an optional deep-learning dependency stack, and less
component-level inspectability, rather than an explicit latent construction
with auditable blocks and neighbour chains.
