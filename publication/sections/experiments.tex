\section{Experiments}
\label{sec:experiments}

The experiments instantiate the evidence framework in
Section~\ref{sec:evaluation-framework}. The primary comparisons ask whether
generated rows avoid trivial reuse, preserve marginal and dependence structure,
and provide useful augmentation for held-out real tasks. The mechanism
diagnostics ask whether the supervised NCA blocks and decoders behave as the
method interpretation suggests. The boundary diagnostics ask whether local
transport and default parameter choices remain stable enough for the restricted
practical-use claim.

Current comparisons include DataFrameSampler plus row bootstrap, independent
columns, Gaussian-copula empirical sampling, and target-stratified column
sampling where a target exists. These baselines define concrete failure modes:
row reuse, broken dependencies, and low-cost marginal sampling. They are not
intended to exhaust the tabular-generation literature.

\subsection{Where the Proposed Method Is Preferred}

The intended preference regime is practical rather than universal. Compared
with row bootstrap, the method is preferred when copying complete records is an
unacceptable default behaviour. Compared with independent column sampling, it
is preferred when cross-column dependence should remain visible in examples.
Compared with heavier tabular generators, it is preferred when a user needs a
small scikit-learn-style object whose fitted latent blocks, neighbour chains,
decoder probabilities, runtime, and failure modes can be inspected directly.
The experiments therefore report where these advantages appear, and they avoid
claiming superiority on datasets or metrics where the evidence does not
support it.

The three named DataFrameSampler setups are shown on one representative
dataset, Adult Census Income. This keeps setup choice separate from the main
cross-dataset evidence: fast, default, and accurate are practical operating
points, not three additional methods to rank across every benchmark.
