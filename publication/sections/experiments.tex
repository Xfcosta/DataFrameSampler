\section{Experiments}
\label{sec:experiments}

The experiments are organized as a claim-specific argument rather than as a
leaderboard. The binding primary claim is practical: DataFrameSampler is
intended to be preferable for low-setup, inspectable synthetic tabular example
generation under limited tuning and moderate data sizes. The binding secondary
claim is mechanistic: the NCA-based latent construction should improve local
categorical consistency enough to make displacement-based generation
meaningful. These claims determine the evidence structure.

\subsection{Claims and Evaluation Structure}

\paragraph{Primary claim: practical use.}
The method is preferable for low-setup, inspectable synthetic tabular data
generation when the workflow values a small estimator, visible mechanisms, and
moderate runtime over fully optimized synthetic-data fidelity.

\paragraph{Secondary claim: mechanism.}
The supervised categorical latent blocks improve local categorical structure,
which makes the local displacement operator more meaningful than applying a
geometric rule directly to one-hot or raw mixed-type columns.

The prepared public dataset suite is designed for coverage rather than
leaderboard breadth. Adult is the running example, Titanic is a small
illustrative case, Bank Marketing provides a mid-sized business benchmark, the
medical datasets exercise small clinical-style regimes, and Covertype adds a
larger public source table with continuous terrain measurements and categorical
terrain context. The controlled datasets isolate high cardinality, rare
categories, dependency structure, and identifier-like columns. Repeated
diagnostics use deterministic row caps so notebook execution remains practical.
The configured DataFrameSampler runs fit NCA blocks on a deterministic 25\%
row sample and then apply the learned projections to all rows; this is the
starting scalability setting used for the experiments rather than a tuned
dataset-specific choice.

Current comparisons include DataFrameSampler plus row bootstrap, independent
columns, Gaussian-copula empirical sampling, and target-stratified column
sampling where a target exists. These baselines define concrete failure modes:
row reuse, broken dependencies, and low-cost marginal sampling. CTGAN is
included only as an optional Adult reference point for a high-capacity global
generative model, not as a fully tuned competitor.

\subsection{Practical Usability: Primary Evidence}
\label{sec:practical-usability}

This block prevents the failure in which a method appears simple in prose but
requires extensive tuning, hidden dependencies, or impractical runtime before
it becomes useful. The instruments are the public API, CLI, runtime table, and
the capped Adult setup comparison. The three named DataFrameSampler setups are
predeclared operating points: fast uses no NCA refinement, no retry budget, and
no calibration; default uses one refinement iteration and a small retry budget;
accurate uses two refinement iterations, a larger retry budget, and calibrated
decoders. They are not treated as three methods to rank across every dataset.

\paragraph{Sensitivity to default parameters.}
We vary $\lambda$, NCA dimension $k$, and iteration count $T$ around the default
configuration and report the corresponding changes in distribution score,
utility lift, and discrimination accuracy. The goal is not to optimize
performance, but to test whether the default configuration is stable under
small perturbations.

The current evidence supports a limited setup claim, not a full
hyperparameter-optimization claim. We do not report equal-budget tuning curves
against all baselines, and therefore we do not conclude that the method is
best under arbitrary tuning budgets. The allowable conclusion is narrower:
under the predeclared default configuration and observed parameter sensitivity,
the method remains usable enough for the intended notebook and example-data
workflow.

\subsection{Data Fidelity and Utility: Primary Evidence}

This block prevents three common failures: generated rows may be near-copies of
training rows, they may preserve isolated marginals while breaking dependence,
or they may look plausible but harm held-out downstream tasks. The instruments
are nearest-neighbour distance tests, real-versus-synthetic discrimination,
distributional summaries, and utility lift on held-out real data. Each metric
supports a different claim: distributional similarity supports resemblance,
discrimination supports non-triviality of the generated distribution, and
utility lift supports task usefulness only where the lift is observed. Formal
definitions of the reported measures are given in
Appendix~\ref{app:measure-definitions}. In the tables, arrows in front of
quality-measure names indicate whether higher values, lower values, or a target
value are preferred.

The full public-dataset tables in Appendix~\ref{app:experiments} give the
numeric summaries and make negative results visible. Row bootstrap can score
well on distributional or utility metrics while copying rows, so it is a
boundary reference rather than a desirable generator. Independent-column
sampling can preserve simple marginals while weakening dependence. DataFrameSampler
is therefore preferred only where the multi-metric pattern avoids these
degenerate alternatives. Where utility lift is weak or negative, the practical
augmentation claim is correspondingly not supported for that dataset.

\subsection{Mechanism Validation: Supporting Evidence}

This block prevents the failure in which useful behaviour is attributed to NCA
or transport without controlled replacement evidence. The first mechanism
instrument tests each categorical block directly. For every categorical column,
the fitted NCA block is used to predict held-out labels and compared with a
majority-class baseline, a same-width PCA projection of the same context, and a
raw-context classifier as a high-capacity reference. These comparisons are
capacity-aware: PCA has the same width as the NCA block, and the raw-context
classifier defines an upper reference for information in the full context.

The second mechanism instrument checks the probabilistic inverse map. Because
categorical reconstruction uses random-forest probabilities, probability
quality must be measured rather than assumed. We therefore report negative log
loss, Brier score, top-class confidence, accuracy, and expected calibration
error. Calibration is supporting evidence for probabilistic reconstruction
quality and boundary evidence for interpreting sampled outputs as calibrated
draws.

The current mechanism evidence is intentionally partial. It tests NCA against
PCA and majority references, and it tests transport indirectly through latent
bootstrap and interpolation diagnostics. It does not yet provide a full
matched-capacity ablation of every decoder and generator replacement. The
allowable conclusion is therefore that the mechanism is supported where the
reported ablations show lift, not that every component is globally necessary.

\subsection{Boundary Conditions and Failure Modes}

This block prevents diagnostic evidence from being read as a guarantee. The
controlled stress tests expose rare-category and high-cardinality regimes; the
frozen-Isomap test examines extrapolative behaviour without claiming manifold
preservation; and the class-rebalancing diagnostic checks an oversampling-
adjacent use case without turning the paper into an imbalance-learning study.

Because the transport operator is adjacent to oversampling-style local
augmentation, we include a limited class-rebalancing diagnostic. This is a
secondary use case. On selected binary target datasets, DataFrameSampler is fit
only on minority-class feature rows, generated rows are labelled as the
minority class, and the augmented training split is evaluated on held-out real
rows. The comparison includes a SMOTE-family reference when the optional
\texttt{imbalanced-learn} dependency is available and a stratified empirical
minority reference.

\subsection{High-Capacity Reference Models}

Modern deep tabular generators are a relevant reference point, but they answer
a different design question. We include CTGAN on Adult to locate
DataFrameSampler relative to a model with a global adversarial objective, a
heavier dependency stack, and lower component-level inspectability. This is not
a leaderboard comparison and it is not a tuned CTGAN study.

\IfFileExists{tables/deep_reference_comparison.tex}{\input{tables/deep_reference_comparison}}{}

The Adult reference run shows a nearby distributional region but a different
tradeoff: CTGAN has a global deep objective and can improve with tuning, while
DataFrameSampler exposes fitted blocks, analogy traces, rejection diagnostics,
and decoder probabilities. The allowable conclusion is design-space
positioning, not superiority over modern deep tabular models.

\subsection{Synthesis: Allowable Conclusions}

The supported conclusion is practical and conditional. DataFrameSampler is
useful for small-to-medium, mixed-type example-data workflows when low setup,
inspectable construction, and dataframe-native outputs matter. It is preferred
over row bootstrap when complete record reuse is unacceptable, over independent
columns when dependence must remain visible, and over heavier generators when
component-level diagnostics are more important than a global objective.
The practical-use conclusion holds under the observed parameter sensitivity and
default configurations reported in Section~\ref{sec:practical-usability}.

The mechanism conclusion is narrower. NCA blocks are supported only where they
improve held-out categorical prediction over majority and same-width PCA
references. Decoder probabilities are usable only to the extent measured by
calibration diagnostics. Frozen-Isomap and convex-hull results are diagnostic
evidence for extrapolative consistency, not a guarantee of manifold
preservation. Class-rebalancing results show that traceable minority examples
can help under a fixed protocol on some datasets, not that the method is a
general imbalance optimizer.

The paper does not support claims of formal privacy, calibrated probabilistic
modelling by default, large-scale optimality, or guaranteed constraint
satisfaction. It supports a bounded design-space claim: an inspectable
analogy-based generator can occupy a useful low-setup point for mixed-type
tabular examples, provided its diagnostics are read together and its failure
modes remain visible.
