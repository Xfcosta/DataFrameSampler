\section{Discussion}

DataFrameSampler 2.0 trades global optimality for a modular and inspectable
construction. Each categorical block is optimized independently, so the
concatenated space has no joint objective. Iterative refinement can improve the
context available to later projections, but it has no monotonic guarantee.

The decoder is also mismatched to the encoder: random forests approximate
$p(C_j \mid z_j)$ independently for each categorical column. Calibration
diagnostics expose probability quality, but they do not impose calibration or
cross-column consistency. Generated rows can therefore violate constraints that
were not learned by the local geometry or the independent decoders.

The generator is intentionally simple. A mutual-neighbour displacement is easy
to inspect, but it is a heuristic, not a formal manifold sampler. This makes the
method suitable for tests, demonstrations, notebooks, and exploratory workflows,
not for regulated privacy-preserving data release.
