# DataFrameSampler

DataFrameSampler generates synthetic rows from a pandas DataFrame or CSV by
encoding each column into discretized bins, sampling in that latent bin space,
and decoding the generated bins back into original column values.

The project is intended as an experimental tabular sampler. Generated data is
not anonymized by design: decoded values are sampled from values observed in
the input bins, so exact source values can appear in the output.

## Principle

Each non-numerical column is converted to a numerical representation in one of
two ways:

1. If `vectorizing_columns_dict` associates numerical helper columns with the
   non-numerical column, a one-dimensional MDS embedding maps those helper
   values to a single value.
2. If no helper columns are configured, category frequency is used as the
   numerical representation.

Each vectorized column is discretized into bins. A row then becomes a vector of
integer bin indexes. New latent rows are generated by selecting a row and a
mutual-neighbor chain, applying a neighbor difference in the latent space, and
decoding the result by sampling original values from the nearest matching bins.

Example: with `name` and `age`, names can be vectorized using age, binned into
age-like regions, sampled in latent space, and decoded by choosing observed
names and ages from the generated bins.

```

```

## CLI Usage

```text
Usage: dataframe-sampler [OPTIONS]

  Generate a dataframe file similar to the input CSV or Parquet file.

Options:
  -i, --input_filename PATH       Path to input CSV or Parquet file.
  -o, --output_filename PATH      Path to CSV or Parquet file to generate.
  -m, --input_model_filename PATH
                                  Path to fit model.
  -d, --output_model_filename PATH
                                  Path to model to save.
  -f, --vectorizing_columns_dict_filename PATH
                                  Path to vectorizing_columns_dict serialized
                                  in YAML.
  -n, --n_samples INTEGER RANGE   Number of samples to generate. If 0 then
                                  generate the same number of samples as input.
  --n_bins INTEGER RANGE          Number of bins.
  --n_neighbours INTEGER RANGE    Number of neighbours.
  -c, --sampled_columns TEXT      Selected columns to generate.
  --anonymize_columns TEXT        Sensitive columns to replace with
                                  OpenAI-generated surrogate values before
                                  fitting.
  --random_state INTEGER          Optional random seed for reproducible output.
  --knn_backend [exact|sklearn|pynndescent|hnswlib|annoy]
                                  KNN backend used for neighbour search.
  --knn_backend_kwargs_filename PATH
                                  Path to backend-specific KNN options
                                  serialized in YAML.
  --embedding_method [mds|pca|incremental_pca|kernel_pca|sparse_pca|truncated_svd|factor_analysis|fast_ica|isomap|lle|spectral_embedding|tsne]
                                  Embedding method for categorical columns
                                  with vectorizing helper columns.
  --embedding_kwargs_filename PATH
                                  Path to embedding-method options serialized
                                  in YAML.
  -A, --auto_config               Use OpenAI to choose omitted vectorizing
                                  columns, sampled columns, embedding method,
                                  and KNN backend.
  -v, --version                   Show the version and exit.
  -h, --help                      Show this message and exit.
```

Example:

```bash
dataframe-sampler \
  --input_filename input.csv \
  --output_filename generated.csv \
  --n_samples 100 \
  --n_bins 10 \
  --n_neighbours 5 \
  --knn_backend sklearn \
  --random_state 42
```

If `--n_samples 0` is used with `--input_filename`, the output row count matches
the input row count.

Parquet input/output is selected by file extension:

```bash
dataframe-sampler \
  --input_filename input.parquet \
  --output_filename generated.parquet \
  --n_samples 1000
```

## Vectorizing Columns

`vectorizing_columns_dict` can be provided as YAML:

```yaml
city:
- country_id
personName:
- age
- country_id
```

This corresponds to:

```python
vectorizing_columns_dict = {
    "personName": ["age", "country_id"],
    "city": ["country_id"],
}
```

## Python Library Interface

The main library entry point is `ConcreteDataFrameSampler`. It returns a fitted
sampler object with a small sklearn-like interface:

```python
sampler = ConcreteDataFrameSampler(
    n_bins=10,
    n_neighbours=3,
    vectorizing_columns_dict=None,
    sampled_columns=None,
    random_state=None,
    knn_backend="exact",
    knn_backend_kwargs=None,
    embedding_method="mds",
    embedding_kwargs=None,
)
sampler.fit(df)
generated_df = sampler.sample(n_samples=100)
sampler.sample_to_file(n_samples=100, filename="generated.csv")
sampler.sample_to_file(n_samples=100, filename="generated.parquet")
sampler.save("model.obj")
```

Arguments:

- `n_bins`: maximum number of bins used to discretize each vectorized column.
- `n_neighbours`: number of nearest neighbors used when building mutual-neighbor
  chains in the latent bin space.
- `vectorizing_columns_dict`: optional mapping from categorical columns to
  numerical helper columns used to embed those categories.
- `sampled_columns`: optional list of columns to generate. If omitted, all
  columns are generated.
- `random_state`: optional integer seed for reproducible output.
- `knn_backend`: nearest-neighbor backend. Supported values are `exact`,
  `sklearn`, `pynndescent`, `hnswlib`, and `annoy`.
- `knn_backend_kwargs`: optional dictionary of backend-specific settings.
- `embedding_method`: embedding/reduction method used for categorical columns
  listed in `vectorizing_columns_dict`. This can be a string, a per-column
  dictionary of strings/objects, or a custom sklearn-like object.
- `embedding_kwargs`: optional dictionary of keyword arguments passed to the
  selected embedding method. This can also be a dictionary keyed by column name.

Complete example:

```python
import pandas as pd
from dataframe_sampler import ConcreteDataFrameSampler

df = pd.DataFrame(
    {
        "age": [21, 22, 35, 36, 48, 49, 63, 64],
        "city": ["Porto", "Porto", "London", "London", "Paris", "Paris", "Rome", "Rome"],
        "spend": [120, 130, 240, 260, 310, 330, 410, 430],
    }
)

sampler = ConcreteDataFrameSampler(
    n_bins=4,
    n_neighbours=3,
    knn_backend="sklearn",
    random_state=42,
)
sampler.fit(df)

generated_df = sampler.sample(n_samples=5)
print(generated_df)

generated_df.to_csv("generated.csv", index=False)
```

Example with categorical vectorization and selected output columns:

```python
import pandas as pd
from dataframe_sampler import ConcreteDataFrameSampler

df = pd.DataFrame(
    {
        "personName": ["Ann Lee", "Bob Kay", "Cam Fox", "Dan Yu"],
        "age": [24, 31, 45, 52],
        "country": ["PT", "PT", "UK", "UK"],
        "country_id": [1, 1, 2, 2],
        "income": [30_000, 42_000, 65_000, 80_000],
    }
)

vectorizing_columns_dict = {
    "personName": ["age", "country_id"],
    "country": ["country_id"],
}

sampler = ConcreteDataFrameSampler(
    n_bins=4,
    n_neighbours=2,
    vectorizing_columns_dict=vectorizing_columns_dict,
    sampled_columns=["personName", "age", "country", "income"],
    embedding_method="pca",
    knn_backend="sklearn",
    random_state=42,
)

sampler.fit(df)
generated_df = sampler.sample(n_samples=10)
```

Saved models are pickle files. Use `sampler.load("model.obj")` only with files
you trust.

## Data And Model Serialization

Generated data can be written as CSV or Parquet. The format is selected from the
file extension:

- `.csv`, `.csv.gz`, `.csv.bz2`, `.csv.xz`, `.csv.zip`, `.csv.zst`
- `.parquet`
- `.pq`

CSV is the most portable and human-readable option. Parquet is usually better
for larger datasets because it preserves column dtypes more reliably, is often
smaller on disk, and is faster to read and write with analytics tools.

Python examples:

```python
sampler.sample_to_file(n_samples=10_000, filename="generated.csv")
sampler.sample_to_file(n_samples=10_000, filename="generated.parquet")
```

The same extension-based IO helpers are available directly:

```python
from dataframe_sampler import read_dataframe, write_dataframe

df = read_dataframe("input.parquet")
write_dataframe(df, "copy.csv")
```

Parquet support requires an installed engine. The recommended extra is:

```bash
pip install ".[parquet]"
```

Fitted sampler models are different from generated data. They contain Python
objects such as encoders, histograms, sklearn reducers, optional custom
embedding objects, RNG state, and neighbor-search configuration. These are saved
with pickle via `sampler.save("model.obj")`; they are not stored as Parquet.
Only load model files you trust.

## Embedding Methods

When `vectorizing_columns_dict` maps a categorical column to one or more
numerical helper columns, DataFrameSampler reduces those helper vectors to one
dimension before binning. The default method is `mds`, matching the original
implementation, but the reducer is configurable.

Supported string values:

- `mds`: `sklearn.manifold.MDS`
- `pca`: `sklearn.decomposition.PCA`
- `incremental_pca`: `sklearn.decomposition.IncrementalPCA`
- `kernel_pca`: `sklearn.decomposition.KernelPCA`
- `sparse_pca`: `sklearn.decomposition.SparsePCA`
- `truncated_svd`: `sklearn.decomposition.TruncatedSVD`
- `factor_analysis`: `sklearn.decomposition.FactorAnalysis`
- `fast_ica`: `sklearn.decomposition.FastICA`
- `isomap`: `sklearn.manifold.Isomap`
- `lle`: `sklearn.manifold.LocallyLinearEmbedding`
- `spectral_embedding`: `sklearn.manifold.SpectralEmbedding`
- `tsne`: `sklearn.manifold.TSNE`

Each string method is instantiated with `n_components=1` unless overridden in
`embedding_kwargs`. If the sklearn class supports `random_state`, the sampler's
`random_state` is passed through unless explicitly overridden.

Simple PCA example:

```python
sampler = ConcreteDataFrameSampler(
    vectorizing_columns_dict={
        "personName": ["age", "country_id", "income"],
    },
    embedding_method="pca",
    random_state=42,
)
```

Embedding-specific options:

```python
sampler = ConcreteDataFrameSampler(
    vectorizing_columns_dict={
        "personName": ["age", "country_id", "income"],
    },
    embedding_method="kernel_pca",
    embedding_kwargs={
        "kernel": "rbf",
        "gamma": 0.05,
    },
)
```

Per-column method selection:

```python
sampler = ConcreteDataFrameSampler(
    vectorizing_columns_dict={
        "personName": ["age", "country_id"],
        "city": ["country_id", "income"],
    },
    embedding_method={
        "personName": "pca",
        "city": "mds",
    },
    embedding_kwargs={
        "city": {"dissimilarity": "euclidean"},
    },
)
```

Custom sklearn-like objects are also supported. The object may implement
`fit_transform(X)`, `fit(X).transform(X)`, or just `transform(X)`.

```python
class FirstColumnProjector:
    def transform(self, X):
        return X[:, :1]

sampler = ConcreteDataFrameSampler(
    vectorizing_columns_dict={
        "personName": ["age", "country_id"],
    },
    embedding_method=FirstColumnProjector(),
)
```

From the CLI, string methods can be selected with `--embedding_method`. Keyword
arguments can be supplied as YAML:

```yaml
# embedding_kwargs.yaml
kernel: rbf
gamma: 0.05
```

```bash
dataframe-sampler \
  --input_filename input.csv \
  --output_filename generated.csv \
  --embedding_method kernel_pca \
  --embedding_kwargs_filename embedding_kwargs.yaml
```

## LLM Auto Configuration

DataFrameSampler includes an optional helper that asks an OpenAI model to inspect
a dataframe profile and recommend:

- `vectorizing_columns_dict`
- `sampled_columns`
- `embedding_method`
- `knn_backend`
- short rationales for the recommendations

Install the optional dependency and set the API key:

```bash
pip install ".[llm]"
export OPENAI_API_KEY="..."
```

Python example:

```python
import pandas as pd
from dataframe_sampler import (
    ConcreteDataFrameSampler,
    suggest_sampler_config_with_openai,
)

df = pd.read_csv("input.csv")

config = suggest_sampler_config_with_openai(
    df,
    model="gpt-4o-mini",
)

print(config["vectorizing_columns_dict"])
print(config["embedding_method"])
print(config["notes"])

sampler = ConcreteDataFrameSampler(
    n_bins=10,
    n_neighbours=5,
    vectorizing_columns_dict=config["vectorizing_columns_dict"],
    sampled_columns=config["sampled_columns"],
    embedding_method=config["embedding_method"],
    knn_backend="sklearn",
    random_state=42,
)

sampler.fit(df)
generated_df = sampler.sample(n_samples=len(df))
```

CLI auto mode:

```bash
dataframe-sampler \
  -A \
  --input_filename input.csv \
  --output_filename generated.csv \
  --n_samples 100 \
  --random_state 42
```

`-A` only fills options that the user did not specify. For example, if
`--embedding_method pca`, `--knn_backend exact`, `-c`, or `-f` are provided,
those explicit values are kept and the LLM recommendation is ignored for those
fields.

For just the mapping:

```python
from dataframe_sampler import suggest_vectorizing_columns_with_openai

vectorizing_columns_dict = suggest_vectorizing_columns_with_openai(df)
```

The helper sends a compact dataframe profile to the OpenAI API: column names,
dtypes, missing counts, unique counts, examples, and simple numeric summaries.
It does not send the full dataframe by default. Review the generated
configuration before using it, especially on sensitive or high-stakes datasets.

LLM-facing usage guidance is also available in:

- `docs/llm_skill.md`
- `docs/vectorizing_columns_prompt.md`

## Sensitive-Value Replacement Before Sampling

DataFrameSampler can optionally replace selected sensitive columns before the
sampler is fitted. This is the recommended order when generated rows must avoid
exact source identifiers:

```text
original dataframe
  -> replace sensitive values with OpenAI-generated surrogates
  -> fit DataFrameSampler on the surrogate dataframe
  -> generate rows from the surrogate dataframe
  -> check generated rows for overlap with original sensitive values
```

CLI example:

```bash
dataframe-sampler \
  --input_filename input.csv \
  --output_filename generated.csv \
  --anonymize_columns personName \
  --anonymize_columns email \
  --n_samples 1000
```

Python example:

```python
from dataframe_sampler import (
    ConcreteDataFrameSampler,
    anonymize_columns_with_openai,
    assert_no_value_overlap,
)

anon_df, report = anonymize_columns_with_openai(
    dataframe=df,
    source_dataframe=df,
    columns=["personName", "email"],
)

sampler = ConcreteDataFrameSampler(
    n_bins=10,
    n_neighbours=5,
    knn_backend="sklearn",
    random_state=42,
)
sampler.fit(anon_df)
generated_df = sampler.sample(n_samples=len(df))

assert_no_value_overlap(df, generated_df, ["personName", "email"])
```

This feature performs sensitive-value substitution with exact and normalized
overlap checks. It is not differential privacy, k-anonymity, l-diversity, or a
guarantee against linkage attacks. It only protects selected columns from exact
source-value reuse.

## KNN Backends

DataFrameSampler builds mutual-neighbor chains in the latent bin space. The
default backend, `exact`, preserves the original behavior by computing all
pairwise distances. This is simple and deterministic, but it has quadratic
memory and runtime costs.

Available backends:

- `exact`: no extra dependencies. Computes the full pairwise distance matrix.
  Best for small datasets and debugging.
- `sklearn`: no extra dependencies beyond the package runtime dependencies.
  Uses `sklearn.neighbors.NearestNeighbors`. This is the recommended first
  faster backend because scikit-learn is already required.
- `pynndescent`: optional approximate nearest neighbors using NN-Descent. This
  is a strong default for larger dense numeric/tabular matrices and supports a
  broad set of metrics. Install with `pip install ".[pynndescent]"`.
- `hnswlib`: optional approximate nearest neighbors using HNSW graphs. This is
  often very fast for large vector search with `euclidean`, `cosine`, or inner
  product metrics. Install with `pip install ".[hnswlib]"`.
- `annoy`: optional approximate nearest neighbors using Annoy indexes. This is
  simple and mature, with useful support for disk-backed indexes outside this
  package. Install with `pip install ".[annoy]"`.

The optional libraries are imported lazily. If a backend is not selected, its
package does not need to be installed. If a selected optional backend is missing,
the error message includes the matching install command.

Approximate backends fall back to scikit-learn for tiny datasets, where index
construction overhead dominates and some ANN libraries are less robust.
When `random_state` is provided to `ConcreteDataFrameSampler`, it is also passed
to `pynndescent`, `hnswlib`, and `annoy` unless explicitly overridden in
`knn_backend_kwargs`.

Python example with PyNNDescent:

```python
sampler = ConcreteDataFrameSampler(
    n_bins=20,
    n_neighbours=10,
    knn_backend="pynndescent",
    knn_backend_kwargs={
        "random_state": 42,
        "n_trees": 10,
        "n_iters": 10,
    },
    random_state=42,
)
```

CLI example with backend options from YAML:

```yaml
# knn_backend_kwargs.yaml
random_state: 42
n_trees: 10
n_iters: 10
```

```bash
dataframe-sampler \
  --input_filename input.csv \
  --output_filename generated.csv \
  --knn_backend pynndescent \
  --knn_backend_kwargs_filename knn_backend_kwargs.yaml
```

The current data representation is dense numeric bin vectors. For this reason,
NN-Descent and HNSW are usually better practical choices than classic
locality-sensitive hashing for this project. LSH-style methods are more useful
for sparse, binary, set-like, or text/Jaccard-oriented data.

## Package Layout

The implementation lives under `src/dataframe_sampler/`:

- `knn.py`: exact and optional approximate nearest-neighbor backends
- `io.py`: CSV and Parquet dataframe input/output helpers
- `llm.py`: optional OpenAI-assisted sampler configuration
- `vectorizer.py`: dataframe-to-numeric vectorization
- `encoding.py`: bin encoding and decoding
- `neighbours.py`: mutual-neighbor estimation and probabilities
- `sampler.py`: latent-space sampling and high-level dataframe API
- `metrics.py`: KL-divergence utilities
- `cli.py`: command-line interface
- `utils.py`: YAML and random-state helpers

The root `dataframe_sampler.py` file is a compatibility wrapper for older
source-checkout usage.

## Safety Notes

Model files are stored with Python pickle. Only load model files that you
created or otherwise trust; loading an untrusted pickle can execute arbitrary
code.

Generated data should not be treated as privacy-preserving synthetic data. The
decoder samples from original bin contents, so output rows can contain exact
values from the source data.
